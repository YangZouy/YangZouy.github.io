openclaw安装及简单了解

简介

OpenClaw是一个开源、可本地部署的自主AI Agent框架，相当于机器人的操作系统与躯体，让机器人拥有长期记忆、操作本地文件并在设备上持续运行的能力。

使用场景

  • 办公自动化：文件管理、邮件处理、日程管理、天气/股票生成日报。
  • 浏览器自动化：数据抓取、表单填写、内容发布。
  • 开发与运维：代码自动生成、Docker自动化运维、跨平台转发消息。

前置要求

  • 操作系统：Windows
  • 运行环境：Node.js 22 及以上版本
  • 工具：Git
  • 网络：需配置代理（梯子），使用 PowerShell（管理员身份）运行

一、安装

npm install -g openclaw@latest

当前安装版本：2026.4.9

二、初始化配置

openclaw onboard --install-daemon

向导中大部分选项可点击 Skip now，稍后进入配置环节。

三、启动网关

若初始化后无法打开 WebUI，需手动启动网关服务。有两种模式：

|—|—|—|
|命令|模式|特点|
|openclaw gateway start|后台运行|通过 Windows 计划任务后台启动，不占用终端窗口；但遇错会静默退出，无报错信息|
|openclaw gateway run|前台运行|进程挂载在当前 PowerShell 窗口，可实时查看日志和错误，不可关闭窗口|

四、核心概念

  • Gateway：后台服务进程，负责与模型通信、执行命令、管理记忆及读写文件
  • WebUI：浏览器端操作界面
  • CLI：命令行工具，用于安装、配置、启停 Gateway 服务
    工作流程：WebUI 输入指令 → Gateway 接收并执行 → 结果返回 WebUI

五、访问地址与Token

http://127.0.0.1:18789/#token=xxx

Token 为登录凭证，也可通过以下命令查看：

cat ~/.openclaw/openclaw.json

六、常用命令

操作 命令
重启网关 openclaw gateway restart
后台启动网关 openclaw gateway start
前台启动网关 openclaw gateway run
查看已接入模型 openclaw models list
重新配置（终端） openclaw configure（建议勿直接编辑 .json 文件）
访问终端界面 openclaw tui

七、模型接入

当前已接入：MiniMax M2.7

八、其它

  1. Skills商店：
  • clawHub：中文不友好
npm i -g clawhub
clawhub install 'skill-name'
  • cocoloop：中文友好
  1. 连接飞书、微信等平台原因：
  • 扩展Agent的输入输出渠道
    • 不用在命令行输入命令，平台中可直接输入
    • 输出内容可直接发送到平台
    • 团队协作——多人在一个群中与Agent进行交互
    • 某些Skill可以直接操作这些平台
